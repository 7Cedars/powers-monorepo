# Safe_Setup

Safe_Setup is a specialized "bootstrap" mandate that deploys a new Gnosis Safe, configures it with necessary modules, and links it to the Powers contract as its Treasury.

## Overview

This mandate simplifies the initial setup of an organization's treasury infrastructure. Instead of requiring multiple separate transactions to deploy a Safe, enable modules, and update the Powers configuration, `Safe_Setup` handles the entire sequence atomically.

Once the setup is complete, the mandate automatically revokes itself to ensure it cannot be reused.

## Configuration

When adopting a Safe_Setup instance, the following parameters are required:

1.  `safeProxyFactory` (address): The address of the Safe Proxy Factory used to deploy new Safes.
2.  `safeL2Singleton` (address): The address of the Safe L2 Singleton implementation (the "master copy").
3.  `allowanceModule` (address): The address of the Allowance Module to be enabled on the new Safe.

## Usage

### Proposing an Action

This mandate does not require any input parameters.

### Execution Flow

Unlike standard mandates, `Safe_Setup` uses a custom execution logic (`_externalCall`) to perform a multi-step process:

1.  **Deployment**
    *   Deploys a new Safe Proxy using `createProxyWithNonce`.
    *   The new Safe is initialized with the Powers contract as its sole owner (`threshold = 1`).

2.  **Configuration (via Callback)**
    *   Constructs a batch of calls to be executed by the Powers contract:
        a.  **Enable Module:** Calls the new Safe to enable the `allowanceModule`.
        b.  **Set Treasury:** Calls `IPowers.setTreasury` to register the new Safe address.
        c.  **Self-Destruct:** Calls `IPowers.revokeMandate` to remove the `Safe_Setup` mandate itself.

3.  **Completion**
    *   The Powers contract executes these calls, finalizing the setup.

## Technical Specifications

### Functions

#### `handleRequest`
```solidity
function handleRequest(...) public pure override returns (...)
```
-   Returns a placeholder call to trigger the internal execution logic.

#### `_externalCall`
```solidity
function _externalCall(...) internal virtual override
```
-   Performs the actual Safe deployment.
-   Constructs the configuration calls.
-   Triggers `IPowers.fulfill` with the generated actions.
